从用户反馈或使用经验中学习，更新项目记忆和开发规范。

学习内容: $ARGUMENTS

## 工作流程

1. 理解用户提供的反馈或经验
2. 判断这属于哪类知识:
   - **Bug修复经验** → 记录到 docs/changelog.md 并更新代码
   - **用户偏好** → 保存到 memory 系统 (feedback 类型)
   - **IHC领域知识** → 保存到 memory 系统 (project 类型)，并考虑是否需要调整默认参数
   - **开发规范** → 更新 CLAUDE.md
3. 执行对应的记录操作
4. 如果涉及代码改进，询问用户是否立即实施
